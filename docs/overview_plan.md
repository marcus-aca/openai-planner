# Personal Finance Dashboard

Scope classification: 4 week MVP

## Overview
Build a single-user web app that imports transactions from CSV, auto-categorizes with editable categories, supports monthly budgets per category, and displays a fast monthly dashboard summary with charts using local-only storage.

## Sections

### S1: Product Requirements & UX Flows
Status: not started

Define MVP user flows, screens, data fields, and acceptance criteria aligned to CSV import, categorization, budgets, and monthly dashboard performance.

### S2: Architecture & Tech Stack
Status: not started

Select a straightforward web stack and define app architecture, local persistence approach, and performance strategy for fast monthly summaries.

### S3: Data Model & Local Storage
Status: not started

Design database schema for transactions, categories, budgets, and optional categorization rules; include migrations and indexing for month-based queries.

### S4: CSV Import Pipeline
Status: not started

Implement CSV upload, validation, parsing, column mapping (minimal), deduplication, and persistence into local DB with feedback to the user.

### S5: Auto-Categorization & Category Editing
Status: not started

Provide baseline auto-categorization and allow users to edit categories per transaction and manage categories; optionally support simple keyword rules.

### S6: Budgeting (Monthly by Category)
Status: not started

Enable users to set monthly budget amounts per category and compute progress based on month-to-date spending for the selected month.

### S7: Dashboard & Charts
Status: not started

Build the monthly dashboard with summary KPIs, budget progress visualization, and category breakdown charts within the 2-second render target.

### S8: Quality, Testing, and Acceptance
Status: not started

Add automated tests for parsing, categorization, budget math, and analytics; define performance checks and MVP acceptance verification.

### S9: Packaging, Local Deployment, and Documentation
Status: not started

Prepare the app for local use with simple run instructions, data storage location clarity, and basic user documentation.

### S10: 4-Week MVP Delivery Plan (Timeline)
Status: not started

Break work into weekly milestones with clear deliverables and a path to hit MVP success criteria within four weeks.
